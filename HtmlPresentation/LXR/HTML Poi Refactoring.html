Rules

CSS - Overview

  • Css set per program code
  • Two css types: one defines layout, one defines image
  • Each page contains its own layout css and image css
  • Layout css shared by all resolutions
  • Resolutions fallback to three category: medium/large/xlarge
  • Different resolution has different image css set
  • Each page contains all of the css, no fetch dynamically

CSS - Media query for resolution (iphone)

  • Use device pixel ratio to determine the device 
  • @media screen and (-webkit-max-device-pixel-ratio:1.4) {
...
}
@media screen and (-webkit-min-device-pixel-ratio:1.5) {	
...
}
  • Why is it different from android?
  • Ios doesn't provide meta tag "target-densitydpi=device-dpi" like android, so the pixels defined in ios will be multipled by device pixel ratio.
    The device pixel ratio in iphone 4/4S is 2.
  • Use "background-size" to shrink the images divided by 2 in iphone 4/4S
  • Ios will then multiple the value by 2 and get the correct image size 
  • .userIcon {
    background:url(large/default_photo_icon_unfocused.png) no-repeat;
    width:32px;
    height:32px;
    background-size:32px 32px;
}

CSS - Why media query images in one css?

  • Use media query calling in external stylesheet will fetch all of the requests, for example: 
  • <link rel='stylesheet' media='screen and (min-device-width: 320px) 
    and (max-device-width: 480px) and (orientation: portrait)' href='css/medium.css' />
<link rel='stylesheet' media='screen and (min-device-width: 481px) 
    and (max-device-width: 960px) and (orientation: portrait)' href='css/large.css' />
  • Will send two requests, one is getting medium.css, one is getting large.css
  • However, if the media query is in one css, browser will dynamically calculate the value based on media query and fetch the correct request

JavaScript - Namespace 

  • function namespace(namespaceString) {
    var parts = namespaceString.split('.'),
        module = window,
        currentPart = '';
		
    for(var i = 0, length = parts.length; i < length; i++) {
        currentPart = parts[i];
        module[currentPart] = module[currentPart] || {};
        module = module[currentPart];
    };

    return module;
};
  • Usage:
